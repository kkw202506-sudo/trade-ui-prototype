# 交易复盘系统 - UI设计文档

## 📋 设计基础

### 配色系统 - A股本地化

**红涨绿跌原则**：遵循中国A股市场习惯，与西方市场相反

```css
/* 盈利色（红色系） */
--color-profit-500: #ef4444;   /* 主盈利色 */
--color-profit-600: #dc2626;   /* 深盈利色 */
--color-profit-100: #fef2f2;   /* 浅背景色 */

/* 亏损色（绿色系） */
--color-loss-500: #22c55e;     /* 主亏损色 */
--color-loss-600: #16a34a;     /* 深亏损色 */
--color-loss-100: #f0fdf4;     /* 浅背景色 */

/* 主色调（蓝色） */
--color-primary-500: #3b82f6;   /* 主要操作色 */
--color-primary-600: #2563eb;   /* 深主色 */
--color-primary-100: #eff6ff;   /* 测背景色 */
```

### 字体系统

**主字体**：Inter, PingFang SC, Microsoft YaHei（优先中文字体）
**代码字体**：JetBrains Mono, Fira Code（价格、日期等数字展示）

```css
字体层级：
--text-xs: 0.75rem    (12px)  辅助信息
--text-sm: 0.875rem   (14px)  正文、表单标签
--text-base: 1rem     (16px)  基础文本
--text-lg: 1.125rem   (18px)  次标题
--text-xl: 1.25rem    (20px)  小标题
--text-2xl: 1.5rem    (24px)  页面标题
--text-3xl: 1.875rem  (30px)  统计大数字

字重：
--font-normal: 400   /* 常规 */
--font-medium: 500   /* 中等 - 强调 */
--font-semibold: 600 /* 半粗 - 标题 */
--font-bold: 700     /* 粗体 - 大标题 */
```

### 间距系统

**基于4px网格的数学比例**

```css
--space-1: 0.25rem   (4px)   微间距
--space-2: 0.5rem    (8px)   紧凑间距
--space-3: 0.75rem   (12px)  内边距小
--space-4: 1rem      (16px)  标准间距
--space-6: 1.5rem    (24px)  区块间距
--space-8: 2rem      (32px)  大间距
--space-12: 3rem     (48px)  章节间距
--space-16: 4rem     (64px)  页面边距
```

### 圆角系统

```css
--radius-sm: 0.25rem   (4px)   按钮、输入框
--radius-md: 0.5rem    (8px)   卡片、表单
--radius-lg: 0.75rem   (12px)  大卡片
--radius-xl: 1rem      (16px)  模态框
--radius-full: 9999px          标签、徽章
```

### 阴影系统

```css
--shadow-sm:  0 1px 2px 0 rgb(0 0 0 / 0.05)      /* 浅层悬停 */
--shadow-md:  0 4px 6px -1px rgb(0 0 0 / 0.1)     /* 卡片悬停 */
--shadow-lg:  0 10px 15px -3px rgb(0 0 0 / 0.1)   /* 浮层、FAB */
--shadow-xl:  0 20px 25px -5px rgb(0 0 0 / 0.1)   /* 模态框 */
```

## 🧱 组件库

### 1. 按钮组件

#### 主要按钮
```html
<button class="btn btn--primary">主要操作</button>
<button class="btn btn--primary btn--sm">小按钮</button>
<button class="btn btn--primary btn--lg">大按钮</button>
```

**使用场景**：
- 主要操作：提交表单、确认操作
- 次要按钮：返回、取消
- 幽灵按钮：辅助操作、轻交互

**交互规范**：
- 悬停：`transform: translateY(-1px)` + 阴影加深
- 禁用：透明度60%，禁用指针事件
- 聚焦：2px主色外框 + 2px偏移

### 2. 表单组件

#### 输入框
```html
<input type="text" class="form-input" placeholder="输入提示">
```

**设计规则**：
- 高度：40px（标准）
- 内边距：12px垂直
- 边框：1px灰色
- 聚焦：主色边框 + 3px主色半透明阴影

#### 文本域
```html
<textarea class="form-input form-textarea" placeholder="多行文本..."></textarea>
```

**最小高度**：100px（可调整大小）

#### 选择器
```html
<select class="form-select">
  <option>选项1</option>
  <option>选项2</option>
</select>
```

#### 表单提示
```html
<div class="form-hint">这是帮助提示</div>
<div class="form-error">这是错误提示</div>
```

## 📱 页面设计

### 1. 首页/看板

**页面结构**：
- 顶部：页面标题 + 欢迎语
- 第一行：4个统计卡片（累计盈亏、胜率、盈亏比、最大回撤）
- 第二行：2列网格
  - 左（2/3）：资金曲线图表
  - 右（1/3）：策略胜率横向条形图
- 第三行：最近交易表格

**关键交互**：
- 统计卡片悬停：阴影加深 + 轻微上浮
- 时间范围切换：3M/6M/1Y/全部按钮组
- 策略条形图：悬停显示详细数据
- 交易表格：点击行跳转详情页

**响应式设计**：
- <768px：统计卡片1列，图表垂直堆叠
- 768-1024px：统计卡片2列
- ≥1024px：统计卡片4列

### 2. 交易录入页面

**页面结构**：
- 顶部：页面标题 + 录入提示
- 左侧（2/3）：表单区域
- 右侧（1/3）：侧边提示卡片

**表单分组**：
1. **基本信息**：股票代码、名称
2. **交易信息**：入场日期、价格、数量、仓位比例
3. **出场信息**（可选）：状态切换、出场日期/价格、手续费
4. **策略标签**：
   - 策略类型（单选）：趋势跟随、突破买入、反弹做多等
   - 入场信号（多选）：均线金叉、量价配合等
5. **入场理由**：文本域 + 示例提示
6. **失误归因**（平仓时显示）：止损未执行、判断失误等

**关键交互**：
- 状态切换为"已平仓"：
  - 出场日期/价格字段必填 + 红色高亮背景
  - 显示失误归因区域
- 策略标签：点击切换，单选互斥
- 表单校验：实时校验 + 提交时统一展示错误
- 目标时间：<60秒完成必填字段

**表单提示示例**：
- "三角形整理突破，量价配合，RSI指标处于合理区间"
- "日线MACD金叉，放量突破前高"
- "基本面驱动：季度净利润超预期"

### 3. 统计分析页面

**页面结构**：
- 顶部：页面标题 + 时间范围选择器 + 导出按钮
- 第一行：4个统计卡片（总笔数、胜率、盈亏比、期望值）
- 第二行：2列图表网格
  - 左：资金曲线（累计盈亏 vs 沪深300基准）
  - 右：月度盈亏柱状图（红涨绿跌）
- 第三行：策略胜率分析卡片

**策略胜率分析设计**：
- 每个策略一行，包含：
  - 策略名称
  - 胜率百分比 + 颜色指示（红胜绿负）
  - 进度条（宽度 = 胜率%）
  - 统计数据：笔数/胜率
  - 平均盈利、最大盈利
- 底部AI分析建议卡片：
  - 主色背景 + 左侧主色边框
  - 基于数据的个性化建议

**时间范围切换**：
- 按钮组：3M / 6M / 1Y / 全部
- 点击切换：高亮当前选项 + 刷新图表

**导出功能**：
- 导出格式：Excel（.xlsx）
- 包含内容：所有统计数据 + 图表截图

### 4. 交易详情页面

**页面结构**：
- 顶部：返回按钮 + 标题 + 操作按钮（编辑/删除）
- 第一行：3列网格
  - 左：盈亏摘要卡片（大号盈亏金额 + 百分比）
  - 右：交易详情卡片（9个字段网格）
- 第二行：策略标签卡片
- 第三行：入场理由卡片
- 第四行：归因分析卡片

**交易详情字段**：
- 入场日期、出场日期
- 入场价格、出场价格
- 交易数量、仓位比例
- 手续费、持仓天数

**归因分析设计**：
- 盈利交易：成功因素分析（绿色背景）
- 亏损交易：问题因素分析（红色背景）
- 中性交易：中性因素（灰色背景）

## 🎨 设计原则

### 1. 简洁高效
- **极简录入**：必填步骤不超过5个操作
- **目标时间**：40秒内完成交易录入
- **减少摩擦**：智能默认值、快捷标签选择

### 2. 数据说话
- **图表优先**：可视化比文字优先
- **实时反馈**：录入后立即计算并展示结果
- **趋势洞察**：通过数据暴露重复性错误

### 3. 克制表达
- **避免诱导**：不使用"赚钱"、"牛市"等词汇
- **中性专业**：保持工具属性，不做投资建议
- **合规边界**：明确区分记录工具与投顾服务

### 4. 本地化适配
- **红涨绿跌**：符合A股用户习惯
- **中文字体**：优先PingFang SC、Microsoft YaHei
- **本土化表达**：使用"涨停"、"跌停"等术语

## ♿ 可访问性

### WCAG AA合规

#### 色彩对比度
- 正常文本：4.5:1（主色500 vs 白色 ✓）
- 大文本：3:1（所有标题 ✓）
- 图形元素：3:1（图标、边框 ✓）

#### 键盘导航
- 所有交互元素可通过Tab键访问
- 焦点指示器：2px主色边框 + 2px偏移
- 逻辑Tab顺序：自上而下、从左到右

#### 屏幕阅读器支持
- 语义化HTML：使用`<nav>`, `<main>`, `<aside>`, `<article>`等
- ARIA标签：图标按钮添加`aria-label`
- 表单关联：`<label>`与`<input>`正确关联

#### 触摸目标
- 最小尺寸：44×44px（按钮、标签、可点击区域）
- 触摸反馈：所有触摸操作有视觉反馈

### 包容性设计

#### 减少动画偏好
- 尊重用户`prefers-reduced-motion`设置
- 默认动画：150-500ms ease（不急促）
- 关键交互：微动画（提升体验，不影响功能）

#### 文本缩放
- 支持200%浏览器文本缩放
- 布局自适应，不出现横向滚动
- 相对单位：rem、%、em（避免固定px）

#### 错误预防
- 清晰标签：每个输入框有明确标签
- 说明文字：表单字段有帮助提示
- 实时校验：输入时即时反馈错误

## 📊 数据可视化规范

### 图表类型

#### 1. 折线图（资金曲线）
- **用途**：展示累计盈亏趋势 + 沪深300基准对比
- **颜色**：
  - 主线：主色蓝（`--color-primary-500`）
  - 基准线：浅灰虚线（`--color-gray-400`）
  - 盈利区域：浅红填充（`--color-profit-100`）
  - 亏损区域：浅绿填充（`--color-loss-100`）
- **交互**：
  - 悬停：显示该点的日期、盈亏金额
  - 图例：点击隐藏/显示基准线

#### 2. 柱状图（月度盈亏）
- **用途**：展示每月盈亏分布
- **颜色**：
  - 正值：红色（`--color-profit-500`）
  - 负值：绿色（`--color-loss-500`）
- **标签**：
  - X轴：月份（1月、2月...）
  - Y轴：金额（¥0、¥5,000...）
- **交互**：
  - 悬停：显示详细盈亏金额

#### 3. 横向条形图（策略胜率）
- **用途**：对比各策略类型的胜率
- **颜色**：
  - 胜率≥50%：红色（`--color-profit-500`）
  - 胜率<50%：绿色（`--color-loss-500`）
- **标签**：
  - 左侧：策略名称
  - 右侧：胜率百分比 + 笔数统计
- **交互**：
  - 悬停：显示详细统计（平均盈利、最大盈利）

#### 4. 气泡图（第二期）
- **用途**：发现最佳持仓周期
- **维度**：
  - X轴：持仓天数
  - Y轴：盈亏百分比
  - 气泡大小：交易数量
- **颜色**：
  - 盈利：红色
  - 亏损：绿色
  - 透明度：50%

#### 5. 饼图（第二期）
- **用途**：失误归因分布
- **颜色**：
  - 止损未执行：红色
  - 判断失误：橙色
  - 执行失误：黄色
  - 追涨杀跌：紫色
  - 仓位过重：蓝色

### 图表库推荐
- **Lightweight Charts**（TradingView）：专业金融图表
- **ECharts**：功能全面，支持交互
- **Chart.js**：轻量级，易于使用

## 🚀 交互规范

### 表单交互

#### 输入校验
- **实时校验**：`input`事件触发，即时反馈
- **提交校验**：提交时统一展示错误
- **错误提示**：
  - 字段下方红色文字
  - 输入框红色边框
  - 错误图标提示

#### 自动完成
- **股票代码**：支持输入时自动补全
- **策略标签**：点击选择，支持快捷键（1-6）
- **日期选择**：使用日期选择器，支持键盘输入

#### 智能计算
- **自动计算字段**：
  - 盈亏金额 = (出场价 - 入场价) × 股数 - 手续费
  - 盈亏百分比 = 盈亏金额 / (入场价 × 股数) × 100%
  - 持仓天数 = 出场日期 - 入场日期
- **触发时机**：
  - 价格/数量变化时：实时计算
  - 平仓时：自动触发归因弹窗

### 导航交互

#### 页面切换
- **加载状态**：异步操作显示loading
- **进度条**：页面顶部细进度条
- **骨架屏**：大数据加载时显示骨架

#### 面包屑
- **格式**：首页 > 统计分析 > 策略胜率
- **可点击**：点击跳转到对应层级
- **当前位置**：灰色，不可点击

### 反馈交互

#### Toast提示
- **成功**：绿色背景 + ✓图标，2秒自动消失
- **错误**：红色背景 + ✗图标，5秒自动消失
- **警告**：黄色背景 + ⚠图标，3秒自动消失
- **信息**：蓝色背景 + ℹ图标，2秒自动消失

#### Modal模态框
- **触发**：平仓归因、确认删除、详情编辑
- **遮罩**：半透明黑色背景（rgba(0,0,0,0.5)）
- **动画**：淡入 + 缩小（从110%到100%）
- **关闭**：点击遮罩、ESC键、关闭按钮

#### 确认操作
- **危险操作**：删除、批量删除、重置数据
- **二次确认**：明确提示风险
- **防误触**：确认按钮需要二次点击

### 动画规范

#### 过渡时间
- **快速**：150ms（按钮悬停、输入框聚焦）
- **正常**：300ms（卡片悬停、页面切换）
- **慢速**：500ms（模态框、展开折叠）

#### 缓动函数
- **ease**：标准平滑（大部分动画）
- **ease-in-out**：进入和退出（模态框、抽屉）
- **ease-out**：快速进入、平滑退出（按钮点击）

#### 微动画
- **按钮点击**：缩放0.95 → 1.0（100ms）
- **标签切换**：缩放1.05（悬停）+ 颜色变化
- **卡片悬停**：上浮2px + 阴影加深
- **表格行**：背景色渐变
